Betingelser i JavaScript

Her er en oversikt over ulike typer betingelser i JavaScript og hvordan de brukes.

1. Hvilke betingelser har vi?

  • if / else if / else: For å sjekke én eller flere betingelser.
  • switch: For å håndtere flere mulige verdier for en variabel.
  • Ternary operator: En kort måte å skrive en if-else.
  • Logical operators: Kombinerer betingelser med &&, ||, og !.
  • Short-circuit evaluation: Sjekker sannhetsverdier raskt med x && y eller x || y.

2. Når ville du brukt en IF-statement?

En if-statement brukes når du trenger å evaluere komplekse eller flere betingelser.

let temperatur = 10;
if (temperatur < 0) {
    console.log("Det er iskaldt ute!");
} else if (temperatur < 15) {
    console.log("Ta på en jakke.");
} else {
    console.log("Det er varmt nok!");
}

3. Når ville du brukt en switch case?

En switch-case brukes når du vet at variabelen har et fast sett med verdier.

let dag = "mandag";
switch (dag) {
    case "mandag":
        console.log("Ny uke, nye muligheter!");
        break;
    case "fredag":
        console.log("Snart helg!");
        break;
    case "søndag":
        console.log("Tid for avslapning.");
        break;
    default:
        console.log("En vanlig dag.");
}

4. Sjekk om du skal ha støvler på

Dette eksempelet sjekker om det regner eller er vanndammer.

let regnerDet = true;
let vanndammer = true;

if (regnerDet || vanndammer) {
    console.log("Du bør ta på støvler.");
} else {
    console.log("Du trenger ikke støvler i dag.");
}

5. Hva er ternary?

En ternary operator er en kortere måte å skrive en if-else.

let alder = 20;
let adgang = alder >= 18 ? "Tillatt" : "Ikke tillatt";
console.log(adgang); // "Tillatt"

6. Hva er nesting?

Nesting betyr at én betingelse er inni en annen. Her er et eksempel:

let vær = "regn";
let temperatur = 5;

if (vær === "regn") {
    if (temperatur < 10) {
        console.log("Ta på støvler og en varm jakke.");
    } else {
        console.log("Ta på støvler.");
    }
} else {
    console.log("Ingen støvler nødvendig.");
}
